# User-admin-system-

Object-Oriented Programming (OOP)

Classes & Objects
- A class is a blueprint used to create objects
- An object is an instance of a class (a real example created from the blueprint)
- Classes group related data (properties) and functions (methods) together
- Objects allow multiple instances with different values

- Example idea:
    - Class = “User”
    - Object = “John, Sarah, Mike”


Encapsulation
- Encapsulation means hiding data inside a class
- Data is not accessed directly from outside the class
- Access is controlled using methods (functions inside the class)
- Helps protect data from being changed incorrectly
- Improves code safety and organisation


Benefits of OOP
- Code is easier to organise and understand
- Reusable structure through classes and objects
- Better control over data using encapsulation
- Reduces bugs by preventing direct data manipulation
